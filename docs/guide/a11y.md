# Accessibility

The accessibility module provides focus management, announcements, keyboard-navigation helpers, media-preference signals, and development-time audits.

```ts
import {
  announceToScreenReader,
  auditA11y,
  prefersColorScheme,
  prefersContrast,
  prefersReducedMotion,
  rovingTabIndex,
  skipLink,
  trapFocus,
} from '@bquery/bquery/a11y';
```

## Focus traps

```ts
const trap = trapFocus(document.querySelector('#dialog')!, {
  initialFocus: '#close',
  escapeDeactivates: true,
});

trap.release();
```

## Live-region announcements

```ts
announceToScreenReader('Profile saved');
announceToScreenReader('Form failed to submit', 'assertive');
```

Use `clearAnnouncements()` when you need to reset the shared live region explicitly.

## Roving tabindex

```ts
const toolbar = document.querySelector('#toolbar')!;
const roving = rovingTabIndex(toolbar, 'button', {
  orientation: 'horizontal',
  wrap: true,
});

roving.focusItem(0);
roving.destroy();
```

## Skip links

```ts
const handle = skipLink('#main-content', {
  text: 'Skip to content',
});

handle.destroy();
```

## Reactive media preferences

```ts
const reduced = prefersReducedMotion();
const scheme = prefersColorScheme();
const contrast = prefersContrast();

console.log(reduced.value, scheme.value, contrast.value);

reduced.destroy();
scheme.destroy();
contrast.destroy();
```

## Audit the current document

```ts
const result = auditA11y();

if (!result.passed) {
  console.table(
    result.findings.map((finding) => ({
      severity: finding.severity,
      rule: finding.rule,
      message: finding.message,
    }))
  );
}
```

The audit API is intended for development-time feedback, not as a replacement for manual accessibility testing.
