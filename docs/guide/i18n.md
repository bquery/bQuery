# i18n

The i18n module provides reactive locale state, interpolation, pluralization, lazy locale loading, and Intl-based formatting.

```ts
import { createI18n, formatDate, formatNumber } from '@bquery/bquery/i18n';
```

## Create an i18n instance

```ts
const i18n = createI18n({
  locale: 'en',
  fallbackLocale: 'en',
  messages: {
    en: {
      greeting: 'Hello, {name}!',
      items: '{count} item | {count} items',
    },
    de: {
      greeting: 'Hallo, {name}!',
      items: '{count} Eintrag | {count} Einträge',
    },
  },
});
```

## Translate messages

```ts
console.log(i18n.t('greeting', { name: 'Ada' }));
console.log(i18n.t('items', { count: 1 }));
console.log(i18n.t('items', { count: 5 }));
```

## Reactive locale changes

`$locale` is a signal, so derived translations update automatically.

```ts
const greeting = i18n.tc('greeting', { name: 'Ada' });
console.log(greeting.value); // Hello, Ada!

i18n.$locale.value = 'de';
console.log(greeting.value); // Hallo, Ada!
```

## Lazy-load locales

```ts
i18n.loadLocale('fr', () => import('./locales/fr.json'));
await i18n.ensureLocale('fr');
i18n.$locale.value = 'fr';
```

## Formatting helpers

```ts
console.log(i18n.n(1234.56));
console.log(i18n.d(new Date(), { dateStyle: 'long' }));

console.log(formatNumber(1234.56, 'de-DE'));
console.log(formatDate(new Date(), 'en-US', { dateStyle: 'medium' }));
```

## Notes

- Messages support nested keys.
- Repeated locale loads are cached.
- Deep message merges are hardened against prototype-pollution keys.
